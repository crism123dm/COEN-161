The C Language

C is one of the closest you can get to a machine's hardware without writing assembly code. It gives you basic programming constructs like conditionals and branches while not managing any memory for you.

C is barebones and easy to make mistakes with. But if you can safely write C, you can safely write any language. Most mainstream languages borrow at least one of the following things from C:

  • The syntax
  • The philosophy
  • The internals

The Philosophy

The UNIX philosophy as stated in The Basics of Unix Philosophy is

Write programs that do one thing and do it well. Write programs to work together. Write programs to handle text streams, because that is a universal interface.

C Programs are meant to do exactly one thing well. It completes a single task to its fullest extent. When you write programs in lab for other classes, they always take in text and spit out text because any program is able to understand text.

The Internals

Some higher level languages (up until the advent of Golang and Rust) compiled down to C. Both Python and JavaScript had interfaces to connect code in those languages to C. Python is even built on top of C, although technically it can be built on top of any language.

The Syntax

The curly brace syntax is a staple of most programming languages. There are other branches, like the LISP based languages and Python which eschews the clutter of punctuation for the density of whitespace. But many languages still take the basic building blocks of C's style.

Even if the system call changes, the structure of the code still remains the same.
